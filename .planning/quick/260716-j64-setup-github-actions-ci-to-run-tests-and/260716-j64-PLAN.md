<task type="auto">
  <name>Task 1: Add GitHub Actions CI workflow</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Create `.github/workflows/ci.yml` (create `.github/workflows/` if missing). Do not use Nix in CI — use `actions/checkout` and `actions/setup-node` with Node 24 to match flake `nodejs_24`.

Workflow requirements:
- Name: `CI` (or equivalent clear name)
- Triggers: `push` and `pull_request` on `main` (include other default branches only if the repo already uses them; otherwise `main` is enough)
- Job on `ubuntu-latest`, with `env.CI: true` (or rely on Actions default `CI=true` — either is fine; do not unset it)
- Steps in order:
  1. Checkout (`actions/checkout@v4`)
  2. Setup Node 24 with npm cache (`actions/setup-node@v4`, `node-version: '24'`, `cache: npm`)
  3. `npm ci` (lockfile present — do not use `npm install`)
  4. Install Playwright browser deps for Chromium only: `npx playwright install --with-deps chromium` (project projects array is chromium-only; do not install all browsers)
  5. `npm run lint` (oxlint)
  6. `npm run build` (`tsc -b && vite build`) — run separately before e2e so typecheck/build failures are obvious in the log
  7. `npm test` (Playwright). Note: `playwright.config.ts` `webServer` already runs `npm run build && npm run preview` on port 4173; the explicit build step above is intentional for clearer failure attribution, even if build runs twice.

Do not add secrets, deploy jobs, or matrix builds. Pin major action versions with `@v4` (or current stable major). Keep the workflow single-job and readable.
  </action>
  <verify>

  </verify>
