# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase goal:** Storyteller can open a phone-first offline app that knows Trouble Brewing roles, setup chart, and night-order data (PLAT-01, PLAT-02).

**Surfaces in scope:** PhoneShell chrome, home/script catalog overview, stub routes `/setup` and `/play`. Not wizard, bag, or coach UI.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding |
| sm | 8px | Compact element spacing, label↔control gap |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section padding, shell horizontal inset |
| xl | 32px | Layout gaps between major blocks |
| 2xl | 48px | Major section breaks (home hero → script card) |
| 3xl | 64px | Page-level top breathing room below safe-area |

**Exceptions:**
- Touch targets: minimum **44×44px** for all tappable controls (PLAT-01 / RESEARCH Pattern 3) — may use 44 even when adjacent spacing is 8/16.
- Safe-area: padding uses `env(safe-area-inset-*)` in addition to scale tokens (not a scale token itself).
- Shell max-width: **40rem (640px)** centered on tablet; full bleed width on phone (RESEARCH PhoneShell).

## Typography

Exactly 4 sizes, 2 weights:

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 16px | 400 | 1.5 | Source Sans 3 |
| Label | 14px | 600 | 1.4 | Source Sans 3 |
| Heading | 20px | 600 | 1.2 | Fraunces |
| Display | 28px | 600 | 1.2 | Fraunces |

**Rules:**
- Product/app name on home uses **Display**.
- Script card title uses **Heading**.
- Supporting sentences, role list rows, stub copy use **Body**.
- Team badges, meta (“22 roles”, “Offline ready”) use **Label**.
- No additional weights (no 500/700). No third font family.

Table-lantern dark shell (matches RESEARCH PWA `theme_color` / `background_color` intent; avoids purple-glow and cream/terracotta AI defaults).

| Role | Value | Usage |

| Secondary (30%) | `#1A1714` | Script card, stub panels, elevated blocks |
| Accent (10%) | `#C4A35A` | Primary CTA fill/text-on-dark emphasis only (see reserved list) |

**Accent must NOT be used for:** links in stub copy, team badges, role names, borders, or decorative icons.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | **Start setup** |
| App name (Display) | **Storyteller Copilot** |
| Home supporting line | Helps you run Trouble Brewing — offline, on your phone. |
| Script card title | Trouble Brewing |
| Script card meta | 22 roles · setup chart · night order |
| Script card status | Available |
| Empty state heading | No scripts loaded |
| Empty state body | Trouble Brewing data did not load. Reinstall or reload the app after connecting once. |
| Error state | Couldn’t read script data. Reload the app. If it keeps failing, reinstall after a network connection. |
| Stub `/setup` heading | Setup |
| Stub `/setup` body | Setup wizard comes in the next phase. |
| Stub `/play` heading | Play |
| Stub `/play` body | Night coach comes after setup. |
| Destructive confirmation | none — no destructive actions in Phase 1 |

**CTA behavior:** “Start setup” navigates to `/setup` (stub). Do not label CTA “Play”, “Continue”, or “Get started”.

**Catalog content policy (Phase 1 home):**
- Show script name, availability, role **count by team** (or total 22), and that night/setup data is present.
- Optional: compact role name list grouped by team — names + team badges only.
- Do **not** dump full Almanac ability text on home. If a one-line ability is stored in JSON, omit from Phase 1 UI or show truncated single line max 80 characters with wrap (prefer omit).
- No official BotC token art.

## Layout & Interaction Contract

### PhoneShell
- `min-height: 100dvh`; `overflow-x: hidden`; safe-area padding on all four sides.
- Single column; horizontal padding `lg` (24px); content `max-width: 40rem`; `margin-inline: auto`.
- No bottom tab bar in Phase 1 (only three routes; home is primary). Stub pages get a text back link: **Back to home** → `/`.
- Primary CTA sits in normal document flow near the script card (thumb-reachable on phone without a sticky footer requirement this phase).

### Home composition (first viewport)
One composition, not a dashboard:
1. Display app name
2. One supporting sentence
3. One script card (Trouble Brewing) + primary CTA
4. No stats strips, schedule chips, or multi-card grids

### Routes
| Path | UI |
|------|-----|
| `/` | Home + TB catalog overview |
| `/setup` | Stub panel with copy above |
| `/play` | Stub panel with copy above |

### Motion (minimal Phase 1)
- CTA: `opacity` + `translateY(4px)` enter on first paint ≤200ms, `prefers-reduced-motion: reduce` → no motion.
- No decorative ambient animation.

### Accessibility
- Visible focus ring on interactive elements (accent on CTA; text-primary outline on Back link).
- Contrast: text primary on dominant ≥4.5:1; accent button text uses `#0B0B0B` on `#C4A35A` fill (≥4.5:1).
- Render catalog strings as text nodes only — never `dangerouslySetInnerHTML` (RESEARCH security).

## UI Considerations

> Populated by the ui-phase UI-consideration probe. Empty/error COPY lives in Copywriting Contract — this section covers state coverage and references those rows.

Applicable state considerations resolved: **8 covered, 2 backstop, 0 unresolved**

| Category | Element(s) | Status | Resolution / Reason |
|----------|------------|--------|---------------------|
| empty | Script catalog (`list-collection`) | ✅ covered | Catalog is bundled; if parse yields zero scripts, home shows Empty state heading/body from Copywriting Contract (not a blank screen) |
| loading | Script catalog | ✅ covered | Data is sync-imported at module init — no loading skeleton; first paint is either populated or error/empty |
| error | Script catalog / Zod load | ✅ covered | Failed `Zod.parse` shows Error state copy; primary CTA hidden until catalog valid |
| populated | Script catalog | ✅ covered | Exactly one script card: Trouble Brewing, status Available, meta line, Start setup CTA |
| partial | Script catalog | ✅ covered | Partial/corrupt role arrays fail Zod entirely → same as error (no half-rendered script card) |
| overflow | Script card / optional role list | 🧪 backstop | Role list (if shown) scrolls inside shell vertically; page never gains horizontal scroll at 390×844 — held-out visual check at verify |
| zero-one-many | Script catalog | ✅ covered | Phase 1 always targets one available script; UI copy is singular (“Trouble Brewing”); zero → empty state |
| long-text | Script title, stub body, role names | 🧪 backstop | Long role names wrap within card; no ellipsis clipping of names; stub body wraps — visual check at verify |
| overflow | PhoneShell / nav | ✅ covered | Shell enforces `overflow-x: hidden`; Back link text wraps if needed |
| long-text | Primary CTA label | ✅ covered | Fixed label “Start setup” — no dynamic truncation |
