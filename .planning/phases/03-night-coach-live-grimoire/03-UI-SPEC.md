# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase goal:** Storyteller can run First Night and Other Nights via next-beat coach — always knowing what to do next and what to say — without a paper night sheet (COACH-01–04, GRIM-03–04).

**Surfaces in scope:** Night ready **Start first night** handoff; `/play` next-beat coach (replace `PlayStub`); night-complete bridge; Demon Info bluff picker; dedicated live grimoire (alive/dead, reminders, bluff edit). Day-phase nomination/vote UI is out of scope.

**Visual inheritance:** Extend Phase 1 `01-UI-SPEC.md` + Phase 2 `02-UI-SPEC.md` table-lantern language (CONTEXT D-19). Do not invent a second theme. Tokens already live in `src/index.css`.

## Spacing Scale

Declared values (must be multiples of 4) — **identical to Phase 1–2:**

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Chip gaps, selected-bluff mark inset |
| sm | 8px | Compact element spacing, label↔control gap, expand chevron gap |
| md | 16px | Default element spacing, beat-card inner padding, grimoire row padding |
| lg | 24px | Section padding, shell horizontal inset (PhoneShell `px-6`) |
| xl | 32px | Gap between beat title block and short prompt |
| 2xl | 48px | Break between grimoire player groups / bluff section |
| 3xl | 64px | Page-level top breathing room on night-bridge only |

**Exceptions:**
- Touch targets: minimum **44×44px** for Next, Back, Grimoire link, Expand, Dead/Alive toggle, bluff chips, reminder chips, Start first night, Start other night, confirm buttons.
- Safe-area: `env(safe-area-inset-*)` via PhoneShell (unchanged).
- Shell max-width: **40rem (640px)** centered on tablet; full bleed on phone.
- Sticky coach footer: height = control row + `md` padding + safe-area-bottom; coach content gets matching bottom padding so expand detail is not obscured.
- Dead/Alive toggle: may be a full-width 44px-tall control per row (not a tiny switch).

## Typography

Exactly 4 sizes, 2 weights — **identical to Phase 1–2:**

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 16px | 400 | 1.5 | Source Sans 3 |
| Label | 14px | 600 | 1.4 | Source Sans 3 |
| Heading | 20px | 600 | 1.2 | Fraunces |
| Display | 28px | 600 | 1.2 | Fraunces |

**Rules:**
- Beat step title (role/procedure name) uses **Heading**.
- Night bridge title (“Night complete”) may use **Display** once.
- Short prompt, expand detail, grimoire truth lines → **Body**.
- Night kind meta (“First night · step 3 of 12”), progress, “Dead”/token chips → **Label**.
- No additional weights (no 500/700). No third font family.

## Copywriting Contract

Voice: table-lantern coach — short, calm, second-person to the Storyteller. Paraphrased procedure coaching only (IP-safe). No Almanac dumps. No official token art. No fake day-phase chrome.

### Night ready handoff (replaces Phase 2 “no CTA” rule)

| Element | Copy |
|---------|------|
| Night ready heading | **Night ready** (unchanged Display) |
| Night ready body (saved) | Assignments are saved. When the table is ready, start the first night. |
| Night ready body (saving) | Saving your assignments… keep this screen open for a moment. |
| Night ready body (error) | Couldn’t save your assignments on this device. Check storage, then try again — nothing here blames you. |
| Primary CTA | **Start first night** |
| Step Back | **Back** (returns to record — Phase 2 pattern) |

**Do not** label the handoff “Play”, “Begin coach”, or “Go to night”.

### Procedural beat titles (Heading)

| Beat id | Title |
|---------|-------|
| dusk | Dusk |
| minion-info | Minion Info |
| demon-info | Demon Info |
| dawn | Dawn |

Wake beats use the **catalog role name** as Heading (e.g. Washerwoman). Sub-label (Body muted): player name in seat — `{Name}`.

### Short / detail prompt policy (Claude discretion — locked here)

| Beat kind | Short prompt (Body, ≤2 sentences) | Detail (expanded Body) |
|-----------|-----------------------------------|------------------------|
| dusk | Settle the table. Lights down when you are ready to wake the first player. | Checklist: confirm all eyes closed; note any day deaths already marked; keep this phone private. |
| minion-info | Wake the Minions. Show them each other and the Demon. | Point silently; do not speak character names aloud if the table can hear. Put heads down when done. |
| demon-info | Wake the Demon. Show them their Minions, then three bluff characters. | Record the three bluffs below before you continue. Bluffs must be good characters not in play. |
| dawn | Dawn. Everyone may open their eyes. | Announce any deaths you decided at night. Do not invent day nominations here — move to table talk. |
| wake (info role) | Wake {Role}. Follow the short procedure, then put them back to sleep. | Use the catalog reminder text as the procedure backbone; end with “ST judges the answer — this app does not auto-resolve.” |
| wake (Imp / kill) | Wake the Demon. They choose a player to kill tonight. | Mark deaths on the Grimoire when resolved. Star-pass: if the Imp kills themself, update who is the Demon on the Grimoire. |
| wake (Spy) | Wake the Spy. Orient the grimoire carefully before you show anything. | Keep ST-private chrome out of sight. Tip the device or cover headers before the Spy looks. (Dedicated shutter view is deferred.) |

Composer may substitute `{Role}` / `{Name}` from beat context. Prefer catalog `firstNightReminder` / `otherNightReminder` paraphrased into detail — never paste scraped Almanac essays.

## Layout & Interaction Contract

### Focal point per primary surface

| Surface | Focal point |
|---------|-------------|
| Night ready | Summary + sticky **Start first night** |
| Coach beat | Beat card (title + short + expand) + sticky **Next** |
| Demon Info beat | Bluff picker inside beat card + sticky **Next** (soft confirm if <3) |
| Night bridge | Display **Night complete** + sticky **Start other night** |
| Live grimoire | Seat-ordered player list (truth + life + reminders) |
| Soft bluff confirm | Issue copy + **Continue anyway** / **Keep editing** |

### Night ready → `/play` (D-05, D-17)

- Add primary sticky footer CTA **Start first night** (accent), matching Phase 2 sticky pattern.
- On tap: set `nightKind: 'first'`, `beatIndex: 0`, `playSurface: 'coach'`, `playStarted: true`; `navigate('/play')`.
- Replace Phase 2 muted “coaching arrives later” body with Night ready body (saved) above.
- Keep summary block; keep **Back** to record.

### `/play` shell (D-01, D-03, D-04)

- Wrap in `PhoneShell`. Route stays `/play` only — `playSurface: 'coach' | 'grimoire' | 'bridge'` in Zustand (not URL).
- **Default landing = coach** (full-screen next-beat). No permanent player strip. No split dashboard.
- Header row: night meta (Label muted, left) + **Grimoire** text link (right, text-primary underline, not accent).
- Main: one secondary-surface **beat card**.
- Footer sticky: optional **Back** text control above full-width **Next** (min-height 48px / `min-h-12`, accent). Ship **Back** as simple `beatIndex` decrement with clamp (D-08 discretion — locked: ship if cursor is index-based).
- **Next** advances cursor; at end of queue → `playSurface: 'bridge'` (not a fake day screen).

### Beat card (COACH-02, COACH-03, D-02)

1. Heading: procedure/role title  
2. Body muted: player name for wakes; omit for pure procedures  
3. Body: short prompt  
4. **More detail** / **Less detail** toggle — **inline expand** reveals detail paragraph(s) in the same card. No Vaul sheet unless detail exceeds ~1200 characters (not expected for TB).  
5. Demon Info only: bluff picker block below short prompt (before or after expand — prefer **below short, above expand**).

Do **not** list the full night sheet with empty “not in play” slots (D-07).

### Bluff picker (GRIM-04, D-10–D-12)

- Show up to three selected slots + chip grid of eligible roles (Townsfolk/Outsider, not in play).
- Tap chip toggles selection; max 3 selected (ignore further taps or deselect-to-replace — prefer: at 3, new tap does nothing until a selected chip is cleared).
- Selected chips use accent fill; unselected use team badge colors.
- Leaving via **Next** with `< 3` → soft `ConfirmDialog` (copy above).

### Night bridge (D-06)

- Full-screen within `/play`; header may keep **Grimoire** link.
- No nomination/vote controls.
- **Start other night** → `nightKind: 'other'`, `beatIndex: 0`, clear `diedTonightIds`, `playSurface: 'coach'`.

### Live grimoire (GRIM-03, D-13–D-16)

- Dedicated full-screen panel (`playSurface: 'grimoire'`), not a thin overlay.
- Header: **Grimoire** + privacy line; **Back to coach** returns to current beat (cursor unchanged).
- Seat-ordered list; each row always shows **ST-private truth**:
  - True role name (team badge)
  - If Drunk: **Drunk · believes {Cover}** on same row (Body muted)
  - If unassigned: **Role not recorded**
- **Dead / Alive** toggle control per row (44px min) — toggles `deadPlayerIds`.
- Reminders: show chips on row; tap **Add reminder** or row → inline picker of `role.reminders[]` + existing chips to clear. Tap player → place/clear (D-15). No global token tray.
- Bluffs section at bottom: show three slots; editable with same eligibility rules as Demon Info.
- Dead players remain listed (truth for ST); Other Night coach omits their wakes via engine (not by hiding the row).

### Partial grimoire safety (D-18)

- Coach builds wakes only from recorded assignments — never invents players.
- If queue empty after Start: Empty coach copy; still allow Grimoire.

### Spy privacy (RESEARCH open Q — locked)

- **No** dedicated obscure shutter view in Phase 3.
- Spy wake detail copy warns to orient privacy; ST covers chrome manually.

### Motion

- Beat enter / surface switch: opacity + `translateY(4px)` ≤200ms; `prefers-reduced-motion: reduce` → instant.
- Expand: height/opacity ≤150ms; reduced-motion → instant show/hide.
- No ambient night animation, no token particle effects.

### Accessibility

- Visible focus rings: accent on primary CTAs; text-primary outline on Grimoire, Back, Expand, toggles, chips.
- Confirm dialogs: reuse Phase 2 `ConfirmDialog` — `role="dialog"`, focus trap, Escape → dismiss (**Keep editing**).
- Contrast: Phase 1 rules unchanged.
- Render all names/prompts as text nodes — never `dangerouslySetInnerHTML`.
- Long player names wrap in beat sub-label and grimoire rows; no horizontal scroll.
- `aria-expanded` on More detail; `aria-current` optional on active beat meta.

### Content policy

- No official BotC token art.
- Coach copy = paraphrased ST coaching + catalog reminder strings only.
- No ASSIST-* auto answers (Empath count, poisoned false info).
- No day-phase UI.
