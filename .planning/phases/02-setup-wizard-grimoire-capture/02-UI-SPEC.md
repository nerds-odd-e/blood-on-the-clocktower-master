# Phase 2 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase goal:** Storyteller can walk the setup wizard, get a legal TB bag, and record who drew which role before night starts (SETUP-01–05, GRIM-01–02).

**Surfaces in scope:** `/setup` multi-step wizard (script → players → difficulty → bag → deal → record → Night ready). Home “Start setup” CTA and PhoneShell chrome unchanged except accent reserved-list expansion. `/play` remains Phase 1 stub.

**Visual inheritance:** Extend Phase 1 `01-UI-SPEC.md` table-lantern language (D-20). Do not invent a second theme.

## Spacing Scale

Declared values (must be multiples of 4) — **identical to Phase 1:**

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding, Up/Down button internal gap |
| sm | 8px | Compact element spacing, label↔control gap, player-row control cluster |
| md | 16px | Default element spacing, step panel inner padding, roster row vertical padding |
| lg | 24px | Section padding, shell horizontal inset (PhoneShell `px-6`) |
| xl | 32px | Gaps between step heading and primary content block |
| 2xl | 48px | Break between bag team groups; Night ready summary sections |
| 3xl | 64px | Page-level top breathing room below safe-area (first step only) |

**Exceptions:**
- Touch targets: minimum **44×44px** for all tappable controls (Add player, Up/Down, Remove player, More, Next step, Back, role chips, Start night, confirm buttons).
- Safe-area: `env(safe-area-inset-*)` via PhoneShell (unchanged).
- Shell max-width: **40rem (640px)** centered on tablet; full bleed on phone.
- Sticky step footer: when present, footer height = control row + `md` padding + safe-area-bottom; content area gets matching bottom padding so last row is not obscured.

| Label | 14px | 600 | 1.4 | Source Sans 3 |
| Heading | 20px | 600 | 1.2 | Fraunces |

**Rules:**

- “Night ready” title may use **Display** once (celebration of handoff).
- Player names, bag role names, help lines, deal coaching body → **Body**.
- Step progress (“Step 3 of 6”), team headers, profile field labels, meta counts → **Label**.
- No additional weights (no 500/700). No third font family.

## Layout & Interaction Contract

### Wizard shell (replaces `SetupStub` on `/setup`)

- Wrap all steps in `PhoneShell`.
- Route stays `/setup` for every step including Night ready (D-17 / D-18). Wizard step lives in Zustand — **not** URL segments.
- **Step progress:** Label text above heading — `Step {n} of 6` for script…record (`n` = 1…6). Night ready hides the fraction and shows no Back-to-wizard escape except browser/history not required — provide **Back** only if returning to record is supported (recommended: Back from Night ready → record).
- **Footer pattern:** Primary CTA full-width 48px min-height, accent fill. **Back** text control left-aligned above or beside CTA cluster (not accent). On players/difficulty/bag/deal/record: both visible. Script step: Back to home link + Continue setup.
- One job per step screen — no bag preview on players; no role recording on bag.

### Focal point per primary surface

Each step has exactly one visual/interaction focus (thumb-primary). Secondary chrome (Back, progress label) must not compete with accent weight.

| Surface | Focal point |
|---------|-------------|
| Script | Confirm card + sticky **Continue setup** |
| Players | Seat roster list + sticky **Next step** |
| Difficulty | Easy / Standard / Hard segment + sticky **Next step** |
| Bag | Private bag list (team-grouped tokens) + sticky **Accept bag** |
| Deal | Deal coaching card + sticky **Continue to record** |
| Record | Player assignment list + sticky **Start night** (picker is secondary until a row is tapped) |
| Soft-gate dialog | Issue list + **Start anyway** / **Keep recording** |
| Remove dialog | **Remove {Name}?** + **Remove player** / **Keep player** |
| Night ready | Display **Night ready** title + summary block (no primary CTA) |

### Script step
- Single confirm card (secondary surface): script title Trouble Brewing, locked note, Continue setup.
- No multi-script picker (only TB).

### Players step (D-01–D-04)
- Inline list; each row: name input + Up + Down + Remove player + More.
- Up on first row disabled; Down on last row disabled (visible disabled state, not hidden).
- **More** expands Experience / Age / Notes inline under the row (no sheet).
- **Add player** full-width secondary outline button below list; disabled at 15 players with muted hint “Maximum 15 players.”
- Next step disabled when count ∉ [5,15]; show count-gate copy. Uniqueness checked **on Next step only** (D-03) — show duplicate error, stay on step.
- Remove player → confirm dialog before delete (**Remove player** / **Keep player**).

### Difficulty step (D-05–D-06)
- Three equal segmented controls in one row (wrap to 2+1 on narrow if needed — prefer single row with equal flex).
- Default **Standard** selected on first visit.
- One-line help under control (Difficulty help copy).

### Bag step (D-09–D-11)
- Full private bag list grouped by team (Townsfolk → Outsider → Minion → Demon), role names as chips using team badge colors.
- Never list `Drunk` as a draw token; if Drunk in play, show cover Townsfolk in bag and a muted setup note e.g. “One Townsfolk token is the Drunk.”
- Short why-this-bag note (difficulty variant) in muted body under the list.
- **No** Regenerate control. **No** manual edit. Forward CTA: Accept bag. Back returns to difficulty (clearing bag on leave per RESEARCH).

### Deal step (D-12)
- Short coaching card only (Deal heading + Deal coaching). No token checklist, no shuffle animation required.
- Continue to record → record step.

### Record step (D-13–D-15)
- Player list in seat order; each row shows name + assigned character name (or Unassigned hint).
- Tap player → role picker panel (secondary surface, inline below row or full-width panel under list — prefer **full-width panel below list** for thumb reach; not a Vaul sheet).
- Picker lists **only remaining bag tokens**, grouped by team chips. Tap assigns and closes picker.
- Tap assigned player → picker with remaining tokens **plus** Clear role (returns token to pool).
- **Start night** always enabled when on this step (soft gate). If `validateAssignments` fails → confirm dialog listing concrete issues; **Start anyway** / **Keep recording**. On clean pass or confirm → Night ready.

### Night ready (D-17)
- Display title **Night ready**; summary: player count, difficulty name, composition counts (TF/Out/Min/Dem), “Assignments recorded”.
- **No** link or button to `/play`. Muted body explains coaching comes next.
- Persist session so Phase 3 can consume.

### Motion
- Step enter: opacity + `translateY(4px)` ≤200ms; `prefers-reduced-motion: reduce` → instant.
- No confetti, no bag-shuffle animation.

### Accessibility
- Visible focus rings: accent on primary CTAs; text-primary outline on Back, Add player, row controls.
- Name inputs: associated labels (visible or `aria-label` = “Player {n} name”).
- Confirm dialogs: `role="dialog"`, focus trap, Escape → dismiss action (**Keep player** on remove dialog; **Keep recording** on soft-gate).
- Contrast: Phase 1 rules unchanged; destructive button text `#F2EDE6` on `#B33A3A`.
- Render all names/notes as text nodes — never `dangerouslySetInnerHTML`.
- Long player names / notes: wrap within row; max notes length enforced in domain (recommend 200 chars) with wrap, no horizontal scroll.

### Content policy
- No official BotC token art.
- Bag/role UI shows character **names** only (plus team grouping).
- Shoulder-surf: bag privacy line is required on bag step; do not add “show to players” mode.
